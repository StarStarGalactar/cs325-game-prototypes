Toboggan Town

  • Originality: Toboggan Town is a never before seen setting ruled over by the ever spectacular and sadistic toboggan brothers. It is impossible for this game to be a retread of any existing game.
  • Technical Merit: The physics code is functional if wonky. There's an actual game over state with the ability to restart the game.
  • Prototype Postmortem: I've learned the basics of Phaser, the resources available to users of Phaser, and some basics of using Github. It was much easier than I was expecting to create a webserver. Implementing the UI was the most difficult part for me.
  • Prototype Assets:
    • Player character sprite was made by OddPotatoGift. Can be found here.
    • Toboggan Brothers and the Toboggan King sprites were self-made
    • Background is a stock photo from Pexels.com. You can find the image here.
  • Prototype Closest Other Game: The code for this example is a mashup from two Phaser examples.
    • Bounce Knock which can be found here. Most of the physics code is taken from this example as well as the basic object initiations.
    • Invaders which can be found here. I adapted the UI and some collision code from this example.
  • High Concept: Have the kings bump into each other fifty times whilst avoiding the horrific toboggan brothers.
  • Theme: The game is a classic post-apocalyptic story, though confined in a small location and set in feudal times where toboggans are very popular.
  • Mandated Variety: The game uses the arrow keys for movement. The style of game is a bullet dodger. There is no randomness in this prototype.
  • Prototype Goal: This game tests basic movement, score tracking, and object collision.
  • Player Experience Goals: It is my hope that players experience the crushing oppression that is the Toboggan Brother's regime.
  • Gameplay: The player can move King PortlyMan in the cardinal directions with the arrow keys. Bumping into King Toboggan reduces the required number of rendezvous by one. Bumping into the Toboggan Brothers is an immediate death and game over. The player wins if they can bump into King Toboggan 50 times. Players can immediately restart the game when they win or lose by clicking within the game window.
  • Strategies: The trajectory, speed, and position of the objects are the same each time so a player could workshop an ideal run. The code dictating the physics is rather rough so it's possible to push both kings against an edge for rapid bumping.
  • Story/Setting/Premise: After his kingdom was decimated by a nuclear attack, King Portlyman goes to Toboggan Town to request aid from his old friend Toboggan King. However, the Toboggan Brothers have already overthrown King Toboggan in a military coup. Now, King Portlyman must meet with King Toboggan multiple times to plan their escape from Toboggan Town while avoiding the gaze of the evil Toboggan Brothers. Will they succeed or meet a knitted hat demise?
  • Target Audience: This game targets fans of toboggans.
  • Play Time: Depending on the player's skill level, play time can range from a few seconds to forever. It's a score-based game and the only endgame is to achieve a highscore.
